Feature Engineering
استخراج الميزات
  • For each timeframe, computes advanced features including:
    • Price-based: returns, log-returns, volatility, momentum, rolling stats
    • Volume-based: moving averages, std, volume imbalance
    • Order book: spread, depth, imbalance
    • Technical indicators: RSI, SMA, EMA, MACD, ATR, Bollinger Bands
    • Market regime, sentiment, and pattern recognition features
  • All features are aligned and merged across timeframes for a single, rich feature vector per sample (often 100+ features).
  • Feature selection and importance analysis is performed to optimize model input.
  • يحسب ميزات متقدمة لكل إطار زمني، مثل:
    • الأسعار: العوائد، العوائد اللوغاريتمية، التقلب، الزخم، الإحصاءات المتحركة
    • الحجم: المتوسطات المتحركة، الانحراف المعياري، اختلال الحجم
    • دفتر الأوامر: الفارق السعري، العمق، الاختلال
    • المؤشرات الفنية: RSI، SMA، EMA، MACD، ATR، بولينجر باند
    • نظام السوق، المعنويات، واكتشاف الأنماط
  • يتم دمج جميع الميزات عبر الإطارات الزمنية في متجه ميزات واحد غني (غالبًا أكثر من 100 ميزة).
  • يتم اختيار الميزات وتحليل أهميتها لتحسين أداء النماذج.

🧠 Model Training & AI Intelligence

Training Pipeline
  1. Fetches and prepares 50,000+ candles per timeframe for each symbol.
  2. Performs data cleaning, normalization, and outlier removal.
  3. Engineers and aligns multi-timeframe features (100+ per sample).
  4. Prepares target labels (e.g., price movement 24 bars ahead, thresholded for signal).
  5. Splits data into training, validation, and out-of-sample test sets (e.g., last 30 days for OOS).
  6. Trains multiple models:
    • Random Forest (robust to noise, interpretable)
    • XGBoost (handles nonlinearity, fast, accurate)
    • LSTM (captures temporal dependencies)
    • Transformer (advanced sequence modeling)
    • Logistic Regression (baseline, interpretable)
  7. Combines models in a weighted ensemble for final prediction (e.g., 40% XGBoost, 30% LSTM, 30% Transformer).
  8. Performs cross-validation and hyperparameter optimization for each model.
  9. Validates on out-of-sample data, reporting accuracy, win rate, drawdown, Sharpe ratio, and more.
  10. Saves the ensemble as ensemble_model.pkl for each symbol.

⚡ Trading Logic & Execution

  1. Every cycle (e.g., every 60 seconds):
    • Fetches latest market data for the symbol
    • Builds the multi-timeframe feature vector
    • Feeds it to the ensemble model for a prediction: BUY, SELL, or HOLD
    • Logs every prediction and decision
  2. If a trade is signaled:
    • Checks all safety flags (dry-run, paper-trading, safe-mode)
    • Executes the order on MT5 (or simulates/logs if in safe mode)
    • Reduces position size if safe mode is active
    • Sets stop loss and take profit based on ATR, volatility, and AI confidence
    • All actions are logged and (optionally) sent to Telegram for instant alerts
  3. Risk management and position sizing are enforced on every trade

🛡️ Safety, Risk Management & Monitoring

Safety Features
  • Dry Run Mode: Simulates all trades, never risks real money.
  • Paper Trading: Trades on a demo account for real-world testing with zero risk.
  • Safe Mode: Reduces all position sizes to the minimum, even in live trading.
  • Pre-launch diagnostics check model integrity, MT5 connection, and system resources before any trade.
  • Fails fast and notifies you if anything is wrong—no silent disasters!
  • All trades and errors are logged for full auditability.
  • Automatic crash recovery and restart logic in the production launcher.
  • Resource monitoring: alerts you if CPU or RAM usage gets too high.
  • Graceful shutdown and cleanup on exit or interruption.
